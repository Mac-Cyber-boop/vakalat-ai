  <name>Task 2: Create PrecedentRetriever class with ranking</name>
  <files>src/citations/retriever.py</files>
  <action>
Add PrecedentRetriever class to retriever.py:

1. **Constructor**:
   ```python
   def __init__(self, vector_store: PineconeVectorStore):
       self.vector_store = vector_store
   ```

2. **retrieve_precedents(legal_issue: str, filing_court: str, top_k: int = 5) -> list[RetrievedPrecedent]**:
   - Step 1: Query Pinecone with `source_type=case_law` filter (if metadata supports it)
   - Step 2: Over-fetch (k * 2) to allow for re-ranking
   - Step 3: Apply ranking algorithm
   - Step 4: Return top_k results

3. **_rank_precedents(results: list, filing_court: str) -> list[RetrievedPrecedent]**:
   - For each result:
     a. Extract court from metadata (try: court, Court_Name, source fields)
     b. Calculate jurisdiction_weight based on court vs filing_court
     c. Calculate recency_weight from year
     d. Compute final_score: `0.5*semantic + 0.3*jurisdiction + 0.2*recency`
   - Sort by final_score descending

4. **_extract_court(metadata: dict) -> str**:
   - Defensive extraction trying multiple field names
   - Default to "unknown" if not found
   - Normalize court names (e.g., "Supreme Court of India" -> "supreme_court")

5. **_extract_year(metadata: dict, content: str) -> int**:
   - Try metadata fields (year, Year, judgment_date)
   - Fall back to regex on content: `\((\d{4})\)`
   - Default to 2000 if not extractable

6. **Court normalization map**:
   ```python
   COURT_NORMALIZATIONS = {
       "supreme court": "supreme_court",
       "supreme court of india": "supreme_court",
       "delhi high court": "delhi",
       "high court of delhi": "delhi",
       # ... other courts
   }
   ```

Handle Pinecone metadata inconsistency defensively (some records may lack source_type filter).
If filter fails, fall back to unfiltered search + post-filtering.
