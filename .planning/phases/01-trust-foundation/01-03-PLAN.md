<objective>
Create SectionValidator class for verifying section numbers exist in actual law and detecting repealed sections.

Purpose: TRUST-04 requires verifying section numbers exist and are not repealed. The current LegalCodeMapper.is_section_valid() only checks if a section is in the mapping dictionary, not whether it exists in the actual law stored in Pinecone. This plan creates true section validation that queries the vector database.

Output: Working SectionValidator that confirms sections exist in the law database and flags repealed/outdated sections with suggested replacements.
</objective>

    2. Create SectionValidationResult model:
       - status: VerificationStatus (VERIFIED, UNVERIFIED, OUTDATED)
       - act_name: str
       - section: str
       - section_text_preview: Optional[str] (first 200 chars of section text if found)
       - is_repealed: bool = False
       - suggested_replacement: Optional[str] (e.g., "Section 103, BNS")
       - validation_time_ms: int

    3. Create SectionValidator class:

       OLD_CODES = {"IPC", "INDIAN PENAL CODE", "CRPC", "CODE OF CRIMINAL PROCEDURE", "EVIDENCE ACT", "IEA", "INDIAN EVIDENCE ACT"}
       NEW_CODES = {"BNS", "BNSS", "BSA", "BHARATIYA NYAYA SANHITA", "BHARATIYA NAGARIK SURAKSHA SANHITA", "BHARATIYA SAKSHYA ADHINIYAM"}

       __init__(self, vector_store: PineconeVectorStore, code_mapper: Optional[LegalCodeMapper] = None):
       - Store vector_store reference
       - Initialize code_mapper (create if not provided)
       - Store normalized act name mappings

       _normalize_act_name(self, act_name: str) -> str:
       - Normalize to canonical form: "IPC", "CrPC", "IEA", "BNS", "BNSS", "BSA"
       - Handle variations: "Indian Penal Code" -> "IPC"

       _is_old_code(self, act_name: str) -> bool:
       - Return True if act_name is IPC, CrPC, or IEA (pre-2024 codes)

       validate_section(self, act_name: str, section: str) -> SectionValidationResult:
       - Start timer
       - Normalize act_name
       - Build Pinecone filter:
         ```python
         # For statutes, use "act" metadata field (from ingest_master.py)
         filter_dict = {
             "$and": [
                 {"source_type": {"$eq": "statute"}},
                 # Match act name variations
             ]
         }
         ```
       - Query: vector_store.similarity_search(query=f"Section {section}", k=3, filter=filter_dict)
       - Check results for section number in content or metadata

       - If found:
         - If old code (IPC/CrPC/IEA):
           - Use code_mapper.map_section() to get new equivalent
           - Return OUTDATED status with suggested_replacement
         - If new code (BNS/BNSS/BSA):
           - Return VERIFIED status

       - If not found:
         - Return UNVERIFIED with reason

       - Calculate validation_time_ms
       - Return SectionValidationResult

       validate_statute_citation(self, act_name: str, section: str) -> VerificationResult:
       - Wrapper that returns VerificationResult (for consistency with CitationVerifier)
       - Calls validate_section internally
       - Maps SectionValidationResult to VerificationResult

    4. Handle the July 1, 2024 transition:
       - IPC/CrPC/IEA sections are VERIFIED but OUTDATED (still valid for pre-2024 cases)
       - suggested_replacement should say "For matters after July 1, 2024, use Section X, BNS"
