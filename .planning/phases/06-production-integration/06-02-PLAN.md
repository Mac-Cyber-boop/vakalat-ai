4. Create handler for general `Exception`:
   - Log with error level: error_type, error_message, path
   - Return 500 with INTERNAL_ERROR code
   - DO NOT expose exception details in response (security)

5. Create `register_exception_handlers(app: FastAPI)` function that registers all handlers:
```python
from starlette.exceptions import HTTPException as StarletteHTTPException
from fastapi.exceptions import RequestValidationError

def register_exception_handlers(app: FastAPI) -> None:
    app.add_exception_handler(StarletteHTTPException, http_exception_handler)
    app.add_exception_handler(RequestValidationError, validation_exception_handler)
    app.add_exception_handler(Exception, general_exception_handler)
```

  <name>Task 3: Add helper functions for common error patterns</name>
  <files>src/errors/handlers.py, src/errors/__init__.py</files>
  <action>
Add utility functions to `src/errors/handlers.py`:

1. `raise_not_found(resource: str, identifier: str) -> None`:
   - Raises HTTPException(404) with message "{resource} not found: {identifier}"
   - Example: raise_not_found("Template", "bail_application/supreme_court")

2. `raise_validation_error(message: str, field: str = None) -> None`:
   - Raises HTTPException(400) with validation error message
   - Example: raise_validation_error("Invalid document type", "doc_type")

3. `raise_export_error(message: str, format: str) -> None`:
   - Raises HTTPException(500) with export error message
   - Example: raise_export_error("PDF generation failed", "pdf")

4. `log_and_raise(logger: structlog.BoundLogger, level: str, exc: Exception, **context) -> None`:
   - Logs exception with context, then re-raises
   - For controlled error propagation with logging
